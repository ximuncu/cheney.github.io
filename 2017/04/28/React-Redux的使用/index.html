React-Redux的使用

次访问 条评论
2017-04-28 reactjs

在做react大型应用时，Facebook提出了Flux架构模式。Redux就是这个思想中非常热门的前端框架实现。

先理解一下flux中的单向数据流，其实很简单，就是

下面这张图应该就很好理解。

在Redux中也是差不多的思路

Store

Store 就是保存数据的地方，整个应用只能有个Store。Store提供了几个方法：

1、getState() 获取当前状态，也就是store在某个时刻的一个快照。对应一个view
2、dispatch(action) 分发action触发改变state
3、subscribe(listener) 每当 dispatch action 时触发
4、replaceReducer(nextReducer) 替换 store 当前用来计算 state 的 reducer。没有使用过

redux 提供createStore方法来生成一个Store。

这里输入参数包括了Reducer以及一个中间件 thunk。thunk是redux异步action的中间件。

Provider

我们创建好了Store，需要绑定到我们的react组件中，但是有个问题是，我们不可能将这个参数从上而下传递，太麻烦了。react-redux 提供了一个Provider组件用于将store传入。这个组件其实使用的是react组件中context属性，使得所有子组件都是可以获取到store。

connect

React-Redux将组件分为两种，UI组件和容器组件。

UI组件是一个纯展示型的组件，而容器组件用于管理数据。而connect方法用于从UI组件生成容器组件。

这里应用了几个方法

bindActionCreators: 给action creators包装dispatch

mapStateToProps: 将state绑定到props上

mapDispatchToProps: 将dispatch方法绑定到props上

绑定完之后在组件中我们就可以获取state数据及dispatch action

Reducer

Store收到Action 以后，必须给出一个新的State，这样 View 才会发生变化。这种 State 的计算过程就叫做 Reducer。

reducer就是一个函数，接收 state 和 action 参数，返回新的state

redux还提供一个combineReducers方法将多个reducer合并成一个
